   - https://note.com/tsunobuchi/n/n19e7d0a63df6
2. 日本特許情報機構「クレームツリー取扱説明書」

3. 日本弁理士会「構造的クレーム作成方法の提案」

## 実装へ反映した原則

- 長い請求項を、比較可能な一つの構成ごとに分ける。
- `Aと、Bと、Cを備える` の列挙は、A、B、Cを別単位にする。
- 原文の語句を言い換えず、接続表現の `と`、`且つ`、全角セミコロンも保持する。
- 独立項は請求対象と固有構成に分け、従属項は親へ追加される固有構成として扱う。
- 請求項間の依存関係は構成単位の分割とは別のグラフとして保持する。
- 自動抽出した構成タイプと特徴語は、人が画面上で修正できるようにする。

## 構成タイプ

| タイプ | 定義 |
|---|---|
| 請求対象 | 装置、方法、プログラム等の請求カテゴリ |
| 要素 | 部品、領域、手段。方法では意味のある一工程を別扱いする |
| 属性 | 要素の絶対的な性質、形状、数値、材料、状態 |
| 関係 | 複数要素間の位置、接続、包含、比、対応関係 |
| 工程 | 方法クレームの意味のある一つの操作 |
| 機能・効果 | 構成により可能になる処理、作用、効果 |

## 現在の境界規則

- 請求対象の終端: `装置であって、`、`方法であって、` 等
- 列挙終端: `Aと、Bと、` 等。ただし `又は`、`若しくは` は分割しない
- 述語完結: `備え`、`有し`、`構成され`、`形成され` 等の後に新しい主語が始まる箇所
- 文末: `。`、`;`、`；`

空白と改行は解析用表示で連結するが、意味を持つ文字は置換しない。自動結果は
法的判断ではなく、人手レビュー用の下書きとする。
